# Snapshot Data

The IA snapshot files live inside the package at
[`src/fundlens/data/snapshot_data/`](../src/fundlens/data/snapshot_data/):

| File | Rows | Source |
|---|---|---|
| `ia_funds.csv` | ~1762 | Cleaned IA fund list with triaged ISIN resolutions |
| `ia_screen_results.parquet` | ~1451 | Pre-computed alpha-screen results |

Bundling the snapshot inside the package (rather than at the repo root) means
it ships inside the wheel and resolves identically under editable installs,
wheel installs, and Streamlit Community Cloud — the loader
(`fundlens.data.snapshot.load_ia_snapshot`) finds it via
`Path(__file__).with_name("snapshot_data")`, the same pattern the rest of the
package uses for `benchmark_map.yaml` and the report template.

## Provenance

Generated by [`scripts/build_snapshot.py`](../scripts/build_snapshot.py) from
the source repo's reports:

- **Funds:** `reports/ia_resolutions_20260708_triaged.csv` (Investment
  Association fund list + Morningstar ISIN resolution + manual triage review)
- **Screen:** `reports/ia_alpha_screen_checkpoint_matched_500_w4.csv` (alpha
  analysis output for matched funds)

Generated: 2026-07-17.

## Contents

Public data only: fund names, management company names (both public), ISINs,
IA sector classifications, and derived alpha metrics. No personal data, no
portfolio holdings of individuals.

## Refreshing

After updating the source data in the private repo:

```bash
python scripts/build_snapshot.py \
    --funds <path-to-triaged-csv> \
    --screen <path-to-checkpoint-csv>
git add src/fundlens/data/snapshot_data/
git commit -m "Refresh IA snapshot"
```

(The default `--out` is `src/fundlens/data/snapshot_data/`.)

## Live data

The screen tab's "Re-run live" button bypasses this snapshot and fetches fresh
data from Yahoo Finance and the factor sources. It is capped at 50 funds per
run on Streamlit Cloud.
